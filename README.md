# Macro Crisis Dashboard

マクロ経済の危機シグナルを、緑・黄・赤で確認するNext.jsダッシュボードです。

現行機能、判定ロジック、データソース、API、運用手順は [システム仕様書](docs/SPECIFICATION.md) を参照してください。

## X自動投稿

市場・ニュース・SEC 13Fを使ったX投稿候補の自動生成、別AIによるファクトチェック、編集長選定、Vercel Cron、DRY_RUN、管理画面を実装しています。初期設定ではXへ投稿されません。

- 管理画面: `/admin/x-automation`
- 運用仕様: [`docs/X_AUTOMATION.md`](docs/X_AUTOMATION.md)
- 環境変数: [`.env.example`](.env.example)

実行前に`DRY_RUN=true`と`AUTO_POST_ENABLED=false`を確認してください。

## ローカル起動

```bash
cp .env.example .env.local
npm install
npm run dev
```

`.env.local` の `FRED_API_KEY` にFRED APIキーを設定してください。未設定またはAPI取得失敗時、FRED指標は「取得不可」と表示されます。

## データ仕様

- HY OAS: `BAMLH0A0HYM2`（FREDの%値をbpへ変換）
- VIX: `VIXCLS`
- 米10年国債利回り: `DGS10`
- BAA-AAA: 同一観測日の `BAA10Y - AAA10Y`
- 初回失業保険申請件数: `ICSA`（件数をKへ変換）
- IG OAS: `BAMLC0A0CM`
- CCC OAS: `BAMLH0A3HYC`（FREDの%値をbpへ変換）
- 米30年国債利回り: `DGS30`
- SOFR: `SOFR` と `DFEDTARL` / `DFEDTARU` を比較

CMBS、Private Credit、PIKなど、公的な即時APIがない構造指標は `src/data/manual-indicators.json` または公表値データで管理します。米国債入札、銀行預金、MMF、Discount Windowなどは無料公開APIから自動取得します。

## 意思決定支援データ

- 市場影響ニュースの最終フォールバック: `src/data/news.json`
- 過去類似局面: `src/data/similar-periods.json`
- 資産別温度感: `src/data/manual-indicators.json` の `_assetTemperatures`

今日の変化、3行市場サマリー、各指標の変化理由、類似局面、30日シナリオは、取得済み指標からルールベースで生成します。LLM APIや売買予測は使用しません。

ニュースはGDELT DOC 2.0 APIとFRB・FDIC・SECの公的RSSを5分ごとに再検証し、鮮度、重大語、安全弁との関連でTOP3を生成します。画面には取得モードを表示し、全ライブ取得が失敗した場合だけ `src/data/news.json` へフォールバックします。

危機条件はFREDの景気後退確率、消費者心理、CFNAI、Sahm Rule、雇用と、FDIC Bank Failures APIを無料で自動取得します。月次統計は「月次公表時更新」であり、5分ごとに新しい値が出るという意味のリアルタイムではありません。有料のMoody'sモデルや民間PMIは使用しません。

```text
GET /api/news
```

## 3D Risk Core

デスクトップではReact Three Fiberによるリスクコア、安全弁ノード、伝播ラインを表示します。スマートフォン、WebGL非対応端末、低スペック環境、`prefers-reduced-motion` 有効時は軽量なCSSゲージへ切り替わります。

```text
GET /api/indicators
```

3Dノードとモバイルの安全弁ボタンを選択すると、対応する指標カードへ移動します。

# Hidden Gems data source

Hidden Gems supports two data modes:

- `DATA_SOURCE=free` or unset: reads `public/data/hidden-gems.json`
- `DATA_SOURCE=fmp`: uses the existing FMP-backed server calculation

The free snapshot is generated by:

```bash
pip install -r scripts/data-pipeline/requirements.txt
npm run data:pipeline
```

`.github/workflows/data-pipeline.yml` runs the Python batch once per day. It
commits a new snapshot only after enough symbols were fetched and the output
passed JSON validation. A failed run leaves the previous snapshot unchanged.
Yahoo Finance data is unofficial. Analyst coverage, orders, and backlog remain
unavailable unless manually supplied or the FMP mode is enabled.
